# AI問い合わせ対応支援アプリ

 / 優先度 / 要約 / 回答案 / 判定理由を生成し、担当者が確認・編集したうえで保存できるようにしています。

単なる「AIに聞いて終わり」のデモではなく、**問い合わせ受付 → AI支援 → 人間確認 → 対応管理 → 履歴確認** までを一連の業務フローとして扱える構成にしています。

---

## 主な機能

### 1. 問い合わせの登録・一覧管理
- 問い合わせを新規登録
- 一覧画面で内容を確認
- ステータス、優先度、カテゴリなどを見ながら管理
- 検索 / 絞り込みで目的の問い合わせを素早く確認

### 2. AIによる問い合わせ解析
OpenAI API を利用し、問い合わせ本文から以下の情報を生成します。

- 分類（カテゴリ）
- 優先度
- 要約
- 回答案
- 判定理由

これにより、担当者が**最初の切り分けや一次回答のたたき台作成を効率化**できる構成にしています。

### 3. AI結果の確認・保存
AIが生成した結果をそのまま確定するのではなく、**人が確認してから保存**する運用を想定しています。

- AI解析結果を画面上で確認
- 必要に応じて内容を修正
- 保存して問い合わせ詳細に反映

### 4. ステータス更新
問い合わせ対応の進行状況を管理できます。

例：
- 未対応
- 対応中
- 完了

業務アプリとして、問い合わせの状態変化を追いやすいようにしています。

### 5. 集計カード
一覧上部に集計カードを表示し、全体状況をひと目で把握できるようにしています。

- 総件数
- 未対応件数
- 対応中件数
- 完了件数
- 高優先度件数 など

### 6. 優先度の視覚化
優先度ごとに色分けして表示し、重要な問い合わせを視覚的に判別しやすくしています。

### 7. 更新履歴（監査ログ風）
問い合わせの更新内容を履歴として保持し、**いつ・何が変わったか**を追えるようにしています。

これは実務でよくある、
- ステータス変更履歴
- 更新記録の確認
- 対応の追跡

を意識した機能です。

## 工夫したポイント

### 1. AIを「業務フローの一部」として組み込んだ
単なるチャットUIではなく、問い合わせ対応業務の流れに沿ってAIを組み込みました。

具体的には、
- 問い合わせを受け付ける
- AIが一次解析する
- 人が確認して保存する
- 対応状況を更新する
- 履歴を追う

という、**実務に近い一連の流れ**を意識しています。

### 2. 「AIに全部任せない」設計にした
実務ではAI出力をそのまま採用しないことが多いため、**人間が確認して確定する前提**で設計しています。

この点により、
- AI活用の現実的な運用理解
- 業務システムとしての設計視点
- 安全性や説明責任を意識したUI/機能設計

を示せるようにしました。

### 3. 視認性を意識した管理UI
問い合わせ管理では一覧性が重要なため、
- 集計カード
- 優先度色分け
- 検索 / 絞り込み
- 詳細画面での情報整理

を入れて、管理しやすさを意識しています。

### 4. 更新履歴を残して実務感を強めた
ステータス変更やAI結果保存の履歴を残すことで、単なるCRUDアプリではなく、**業務で使う管理画面らしさ**を出しています。
